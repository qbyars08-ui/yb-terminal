What this is

yb-desk is a portfolio desk and an agent office that runs entirely free on GitHub. The dashboard is a static site served by GitHub Pages. The agents are GitHub Actions. No servers, no database, no monthly bill.

The top of the desk shows Quinn's real book, the Young Bull positions and the thesis behind each one. Below that you can build your own desk. The office shows the agents that keep the whole thing running, and the reports they write.

Your data stays in your browser

When you add positions to Your Desk, they are saved to your browser's localStorage under the key ybdesk.portfolio.v1. That is it. Nothing is uploaded, nothing is sent to a server, no account is required. There is no server to send it to.

  • Add, remove, import, and export positions, all client side.
  • Clear your browser storage and your desk is gone, only from this device.
  • Export to CSV any time to keep your own copy. The header is exactly ticker,shares,cost_basis.
  • Prices come from a small static file the price agent refreshes, so P&L updates without touching your holdings.

How the agents work

The office is a set of small jobs that run on GitHub Actions. Some run on a schedule, one runs on demand when you ask it to.

  • Price agent keeps the tape fresh, refreshing quotes every 30 minutes during market hours.
  • Desk report writes the daily desk report after the close, around 5pm ET.
  • Watchdog flags any desk ticker that moved more than 5 percent, around 4:30pm ET.
  • Research agent runs on demand. Open a GitHub issue titled research: TICKER and it replies in the thread.

Each agent writes its output back into the repo as plain JSON or Markdown. The dashboard just reads those files. That is the whole trick: the site is static, the data is files, and the files get updated by Actions.

Run your own fork

The whole thing is meant to be forked. To run your own copy:

  • Fork qbyars08-ui/yb-desk on GitHub.
  • In the fork's settings, turn on GitHub Pages and point it at the docs/ folder on main.
  • Replace data/book.json with your own book, or start empty and just use Your Desk.
  • Enable Actions to bring the office to life. Until you do, the office shows every agent as not yet run.
  • Zero dependencies. Plain Node built-ins for the agents, plain HTML, CSS, and JavaScript for the site. No frameworks, no build step.

The rules of the desk

This is a tool for thinking, not a recommendation engine. Quinn's book is shown so you can see the reasoning, the thesis, the variant perception, and the kill vectors that would prove each position wrong. That is the point. Read the kill vectors before you copy anything.
